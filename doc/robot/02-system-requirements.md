# 系统需求（机器人整体：硬件 + 网络 + 运维）

## 1. 系统范围

系统 = 机器人本体（底盘/音频/电源/网络/计算） + 现场网络与部署环境 + 本项目软件（前后端 + 对接 RAGFlow/ASR/TTS）。

## 2. 硬件需求

### 2.1 音频输入
- 近场/远场麦克风（建议阵列），支持回声消除（AEC）与降噪
- 采样率建议 16k/48k；软件侧统一转换到 16k mono 供 ASR

### 2.2 音频输出
- 扬声器足够响度，避免失真；建议外放 + 自动增益控制（AGC）
- 若有“私密模式”，可支持耳机/定向音柱（可选）

### 2.3 计算与存储
- 本地工控机/边缘设备：支持稳定跑浏览器前端与本地后端代理
- 存储用于缓存日志与必要的运行数据（不存储敏感语音，除非明确授权）

### 2.4 网络
- 优先有线；否则双频 Wi-Fi + 4G/5G 备份
- 断网/弱网时软件需可降级（见 5）

## 3. 安全与合规
- 明示录音提示（语音交互指示灯/提示语）
- 默认不做人脸识别/不采集身份；语音数据只用于实时识别，不长期存储
- 后台管理访问需鉴权（至少内网 + 口令/令牌）

## 4. 现场运行与运维
- 开机自启与自恢复（崩溃自动拉起）
- 远程升级/回滚（版本号可追踪）
- 运行监控：CPU/内存/网络、ASR/TTS/RAG 调用成功率、关键延迟

## 5. 容错与降级策略（系统级）
- 网络不可用：切换到本地“脚本讲解模式”（P0 展区概述 + 关键亮点）
- ASR 失败：提示重试；支持“点击输入”作为备用通道
- TTS 不可用：文本照常显示，允许点击“重播”
- 过热/电量低：自动提示并进入省电/待机
