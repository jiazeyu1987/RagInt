# 验收指标与测试计划（展厅机器人）

## 1. 验收指标（建议）

### 1.1 体验类
- 首句可听时间（点击提交→听到声音）P50 < 3s，P90 < 6s
- 连续讲解跨站停顿：P50 < 0.8s（以现场主观可接受为准）
- 打断响应：触发打断后 300ms 内停止播放；后端旧 request 不再继续产生 TTS

### 1.2 稳定性
- 连续运行 8 小时无崩溃；内存稳定（增长 < 20%）
- 连续 50 次提问无“白噪音/断流”可复现问题

### 1.3 准确性
- 中文 ASR 在展厅噪声下：关键实体词（产品名/展区名）识别准确率达到可用阈值（现场抽样统计）
- RAG 回答：对知识库内容有引用或可追溯；不确定时能明确表达

## 2. 测试用例（最小集）

### T1 连续讲解
- 启动连续讲解，从第 1 站到第 10 站结束
- 观察：是否重复过渡话术、停顿是否明显、是否卡顿

### T2 打断/继续
- 在第 3 站讲解中插话提问（3 次）
- 每次：确认旧声音停止、旧后端 request 被 cancel、回答完可继续

### T3 多人队列
- 提交 5 个普通问题 + 1 个高优先级问题
- 观察：高优先级是否插队，队列是否可控

### T4 弱网/断网
- 降低带宽/断开网络
- 观察：是否进入降级提示；能否恢复

### T5 音频兼容
- Chrome/Edge 不同设备测试录音 MIME（webm/ogg/mp4）
- 观察：是否能正常转 wav 并识别
