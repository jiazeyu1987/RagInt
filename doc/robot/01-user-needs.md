# 倒推用户需求（展厅机器人）

> 说明：本文用于快速对齐“本期要做什么、做到什么程度”。范围刻意收敛，确保项目能按期落地。

## 1. 目标与边界

- **目标**：在同一个房间内，机器人带领参观者按既定站点参观并讲解；支持语音播报；在观众提问时可中断当前讲解并进行问答；问答结束后可继续讲解。
- **边界（本期不做/弱化）**：
  - 不自研 SLAM/自主建图：房间小、路线固定，优先采用预设点位/第三方底盘能力；
  - 不做开放域陪聊：只覆盖展厅讲解、展区相关问答、带路到站；
  - 不做人脸/身份识别：不采集、不存储涉及身份的视觉信息；
  - 不扩展多语种、大屏联动、统计分析等增强项。

## 2. 使用角色与关注点

### 2.1 观众
- 需要：讲解清晰、节奏连贯；提问能被及时响应；
- 主要痛点：播报卡顿或白噪音；打断后仍播放上一段内容；到站不准导致讲解与位置错位。

### 2.2 讲解员/运营
- 需要：一键开始/暂停/继续/跳站；站点顺序与每站讲解时长可配置；能看到机器人当前站点与移动状态。
- 主要痛点：现场异常难定位（语音/知识库/移动）；无法快速介入恢复流程。

### 2.3 运维
- 需要：关键链路可观测（阶段耗时、失败原因、重试次数）；网络不稳时能降级继续提供讲解；日志可追踪。
- 主要痛点：外网抖动导致核心能力不可用；长时间运行后出现性能衰减但缺少定位手段。

## 3. 典型使用流程

### S1：开始巡讲
- 操作员点击“开始讲解”
- 机器人移动至第 1 站并开始播报讲解
- 讲解结束后移动到下一站并继续讲解

期望：跨站停顿尽量短；过渡语自然，避免重复“欢迎来到/接下来我们来到”等固定话术。

### S2：观众提问/打断
- 讲解过程中观众提问（语音触发）
- 系统立即中止当前播报与生成流程，切换到问答
- 回答结束后可继续讲解（从当前站补充或直接进入下一站）

期望：打断需立即生效，旧流程必须停止；继续讲解时能自然承接。

### S3：走到指定展区
- 按预设站点位姿移动（室内小范围，重点是稳定到位）
- 到位后播报提示并开始讲解
- 移动失败可提示并等待人工辅助/手动恢复

期望：移动可控且有反馈；到站准确即可，不追求复杂定位能力。

## 4. 本期交付

- **讲解与移动联动**：支持开始/暂停/继续/下一站/上一站/跳站；到站自动播报。
- **全链路中断**：新问题进入时，旧的流程必须可取消，避免“旧音频继续播放”。
- **语音输入可用**：中文识别为主；识别结果回填输入框，允许人工确认后再提交。
- **语音播报稳定**：支持流式播报，控制白噪音/卡顿；失败时至少保留文本输出与重播能力。
- **内容可信**：以知识库/脚本为准；不确定时明确说明并引导人工确认。
- **状态可视化**：可展示当前站点/移动状态，以及关键耗时）。

## 5. 明确不做

- 多人排队/优先级插队等复杂队列机制（本期先保证单人/单问题稳定）。
- 开放域闲聊与情感陪聊（只覆盖展厅相关问答）。
- 多语种、大屏联动、统计分析等增强需求（后续视资源再评估）。
