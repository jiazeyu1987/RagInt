# 瑛泰医疗智能能力建设项目立项书（修改建议）

来源：`doc/BP/瑛泰医疗智能能力建设项目立项书.md` + `doc/BP/背景.txt`

## 1. 面向审批链的“叙事与呈现”建议（总经办 → CFO → 董事长）

- 建议在正文最前新增“**执行摘要（1 页）**”：用 5–8 条要点把“为什么现在做、做什么、不做什么、分几阶段、每阶段封顶多少钱、到什么程度停/继续”讲清楚，让总经办与 CFO 快速完成判断。
- 建议增加“**本次提交请求的决策事项**”：明确本次只请求批准阶段一（+可选阶段二的预算预留/方案论证），其余阶段必须通过阶段评估再上会，降低 CFO 与董事长的心理负担。
- 董事长对 AI 兴趣和期望较大：阶段一“展厅 AI”可以保留为抓手，但建议在摘要里明确其作用是“快速可见成果 + 统一认知 + 为后续沉淀知识与治理机制打底”，避免被解读为“只做展示”。

## 2. 关键内容缺口与补强建议（结合 `背景.txt`）

### 2.1 财务模型需要“更细、更可审”
现有第八章给了预算区间与止损口径，但 `背景.txt` 提到“财务模型较简单、未细分人力/物料/一次性投入、未区分本地部署软硬件与人工成本、阶段四要考虑本地部署”。建议补齐：

- **按成本性质拆分**（建议直接给表格，便于 CFO 审阅）：
  - 一次性：需求与方案/实施服务、知识库整理与结构化、系统开发集成、展厅机器人硬件与部署、安全与审计改造、（阶段四）本地化部署软硬件与机房/云资源初始化等。
  - 持续性：模型/算力调用、运维与升级、知识维护与内容治理人力、权限与审计运营、供应商支持/服务费等。
- **按部署形态拆分**（尤其阶段四）：公有云 API / 私有化（集团私有云）/ 本地化（GPU 与存储等）三种方案的 TCO（年度运维）对比与适用边界。
- **把“用人成本 30–50 万/年”写进测算逻辑**：不建议写“替代岗位”，但可以写“避免新增编制/缩短培养周期/减少重复投入”，并用“等价 FTE”做口径说明，帮助 CFO 将投入与人力成本对齐理解。

### 2.2 阶段五缺少预算口径与运行机制的“财务落点”
当前预算表到阶段四结束。建议新增：
- “阶段五（体系化运行）年度 OPEX 口径”：至少给出“保底运行成本范围 + 可选增强项”，并明确与年度预算的衔接方式（纳入信息化/专项预算或按部门分摊等）。

### 2.3 缺少“可验收的指标”，不利于阶段评估与止损
现有第七章给了评估维度，但建议把“可量化验收指标”前置到各阶段，形成“交付物 + 指标 + 验收人”的结构，例如：
- 使用率（覆盖人群/周活/月活）、问题解决率（一次命中/需人工转接比例）、知识覆盖度（制度/流程/模板覆盖清单）、检索耗时下降、培训满意度等。
- 阶段一可用“演示稳定性 + 讲解内容标准化 + 常见问答准确性（抽检）+ 审计日志可用”作为验收要点。

## 3. 分阶段内容的“收敛与落地”建议

### 3.1 阶段一（展厅 AI）建议补齐交付边界
- 明确交付物：讲解稿结构化、知识库范围清单、问答知识来源清单、语音方案（TTS）与离线兜底策略、演示脚本、权限与审计最小闭环。
- 明确“不做事项”里建议再加一条：对外口径与宣传内容需经总经办/品牌审核，避免“AI 输出即对外承诺”。

### 3.2 阶段二（人事/培训）建议补齐“内容治理机制”
建议新增小节说明：
- 内容责任人（制度/流程/课程的 owner）与更新流程（谁提、谁审、多久更新一次）。
- 输出免责声明模板（AI 辅助解释，不构成制度发布；以正式文件为准）。

### 3.3 阶段三（多部门扩展）建议“先聚焦、再复制”
当前覆盖部门较多，建议把阶段三拆成“第一批 2–3 个部门”与“第二批滚动扩展”，并把“接入门槛”写清楚（资料齐备程度、owner 是否到位、权限边界是否明确），避免范围失控。

### 3.4 阶段四（子公司推广）建议显式纳入“本地化部署”方案
结合 `背景.txt`，建议在阶段四新增：
- 本地化部署目标与边界（哪些数据/场景必须本地，哪些可以继续云/私有云）。
- 软硬件清单与一次性投入/折旧口径建议（由财务最终定）。
- 安全合规要求：网络隔离、账号体系、日志审计、灾备与应急预案。

## 4. 风险与合规章节的加强点（在现有第六章基础上微调即可）

- 增加“数据分级与准入清单”：哪些数据允许入库、哪些禁止（含个人信息、敏感经营信息、技术秘密等的例示）。
- 增加“模型输出风险控制的工程措施”：引用来源显示（可追溯）、高风险问题强制转人工、回答置信度/引用不足提示、敏感词/越权拦截策略。

## 5. 文字与格式层面的可快速修改点（提升专业度）

- 前言处“结合管理层与管理层判断”建议改为“结合管理层判断/决策”。
- 阶段三“实施原则与边界”处存在不必要的空行，建议统一排版。
- 阶段三标题里的反斜杠（如“注册服务中心\”）建议改为正常换行或列表格式，避免 Markdown 渲染差异。
- “不虚报,不编造数据”建议统一为中文标点“，”，并调整为更正式表述（如“所有结论与数据需有来源文件支撑”）。
